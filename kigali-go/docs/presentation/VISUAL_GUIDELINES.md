# Visual Guidelines for Presentation Slides

## Design Principles

### **Color Scheme (KigaliGo Brand)**
- **Primary Blue:** #1E90FF (Transport/Technology)
- **Secondary Green:** #2EB872 (Growth/Safety)
- **Accent Yellow:** #FFC857 (Energy/Warning)
- **Neutral Gray:** #64748B (Text)
- **White:** #FFFFFF (Background/Clarity)

### **Typography**
- **Headings:** Poppins Bold, 36-48pt
- **Subheadings:** Poppins Semibold, 28-32pt
- **Body Text:** Inter Regular, 24-28pt
- **Captions:** Inter Regular, 18-20pt

**Rule:** Nothing smaller than 18pt

---

## **Slide-by-Slide Visual Recommendations**

### **SLIDE 1: Title**
```
Layout: Centered
Background: Gradient (Blue to Green)
Elements:
  - KigaliGo logo (top center, large)
  - Project title (36-48pt)
  - Team names (24pt, list)
  - University logo (bottom corner)
Images: Hero illustration (transport scene)
```

### **SLIDE 2: Problem Statement**
```
Layout: Split screen
Left: Statistics (large numbers, bold)
Right: Relevant image (Kigali traffic/bus stop)
Elements:
  - 3-4 key statistics (use icons)
  - Bar chart or infographic
  - Citation text (small, bottom)
Color: Red/Orange accents for urgency
```

### **SLIDE 3: Existing Solutions**
```
Layout: Table or comparison cards
Elements:
  - 3 columns (Google Maps, Tap&Go, Local Apps)
  - Icons for each solution
  - ✓ Strengths (green)
  - ✗ Weaknesses (red)
  - Gap highlighted in yellow box
Minimal text: 5-7 words per bullet
```

### **SLIDE 4: Solution Overview**
```
Layout: Hero image + text overlay
Background: App screenshot or mockup
Elements:
  - Value proposition (large, centered)
  - 5 key points with checkmarks
  - Simple architecture diagram
  - Phone mockup showing app
Keep it aspirational and clean
```

### **SLIDE 5: Features Part 1**
```
Layout: 3-column grid
Each Feature:
  - Icon (top)
  - Feature name (bold, 28pt)
  - 2-line description
  - Benefit metric (colored box)
Images: Use the SVG illustrations created:
  - route-planning.svg
  - map-illustration.svg
  - fare-calculator.svg
```

### **SLIDE 6: Features Part 2**
```
Layout: 2-row grid (3 features)
Elements:
  - Safety icon (shield)
  - Language icon (globe)
  - Offline icon (cloud with slash)
  - Screenshots of each feature
Emphasis: Visual simplicity
```

### **SLIDE 7: Technical Architecture**
```
Layout: Diagram-focused
Elements:
  - Simple flow diagram (use arrows)
  - Technology stack icons
  - Color-code layers:
    * Frontend (blue)
    * Backend (green)
    * Database (yellow)
Avoid: Too much technical jargon
Keep: Simple, visual
```

### **SLIDE 8: Impact**
```
Layout: Dashboard style
Elements:
  - Big numbers (48pt+) in colored boxes
  - Before/After comparison
  - User testimonial (quote format, italics)
  - Small chart showing improvement
Use: Green for positive metrics
User photo: Silhouette or avatar (privacy)
```

### **SLIDE 9: Roadmap**
```
Layout: Horizontal timeline
Elements:
  - 4 phases with dates
  - Icons for each phase
  - Progress bar showing "We are here"
  - Key milestones in boxes
Colors: Gradient from present (full color) to future (lighter)
```

### **SLIDE 10: Team & Closing**
```
Layout: Grid (team) + Contact info
Elements:
  - Team photo (if available) or names with roles
  - Contact details in footer
  - QR code to demo/repo
  - Social media icons
  - "Thank You" + "Questions?" (large)
Background: Clean, professional
```

## **Universal Slide Rules**

### **DO:**
- ✓ Use high-contrast text (dark on light or light on dark)
- ✓ Limit to 5 bullet points maximum per slide
- ✓ Use icons and images liberally
- ✓ Leave white space (30% minimum)
- ✓ Align everything consistently
- ✓ Use consistent spacing (multiples of 8px)

### **DON'T:**
- ✗ Put paragraphs on slides
- ✗ Use more than 3 fonts
- ✗ Use low-resolution images
- ✗ Clutter with unnecessary decoration
- ✗ Use animations (unless subtle)
- ✗ Include citations in large text (footer only)

## **Accessibility Checklist**

- [ ] All text readable from 10 feet away
- [ ] Color blind friendly (use shapes + colors)
- [ ] High contrast ratios (4.5:1 minimum)
- [ ] No information conveyed by color alone
- [ ] Alt text prepared for images (for materials)
- [ ] Consistent slide navigation

## **Export Settings**

### **Final Presentation File:**
- **Format:** PDF (most compatible)
- **Resolution:** 1920x1080 (16:9 ratio)
- **File size:** Under 10MB (compress images if needed)
- **Fonts:** Embedded or converted to outlines
- **File name:** `TeamName_KigaliGo_Presentation.pdf`

### **Backup Format:**
- PowerPoint (.pptx) with embedded media
- Individual slide images (PNG, high-res)
